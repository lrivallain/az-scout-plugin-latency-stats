Methodology
P50 median RTT Cloud63 RTT = A→B + B→A
  • Azure Docs: values are sourced from Azure Network Latency where Microsoft measures inter-region round-trip latency using internal backbone probes sampled every minute, then publishes monthly P50 (median) RTT values over a rolling 30-day window. Microsoft notes that updates are periodic (typically every 6 to 9 months) and not all public regions are listed until data is available.
  • Cloud63: measurements are one-way, so we compute P50 per direction, then derive RTT by summing P50(A→B) + P50(B→A). If one direction is missing, latency is shown as unknown.
